# 模型速查表 —— 比赛时选模型的决策指南

## 一、拿到题目先判断：什么类型的问题？

看题目前 30 分钟，先确定题目属于以下哪一类：

| 问题特征 | 问题类型 | 推荐模型 |
|---------|---------|---------|
| "评价""排序""选最优""哪个好" | 评价类 | AHP / TOPSIS / 熵权法 |
| "预测""趋势""未来XX年" | 预测类 | 回归 / 灰色预测 / 时间序列 / 神经网络 |
| "最大化""最小化""分配""调度" | 优化类 | 线性/整数/非线性规划、遗传算法 |
| "分类""识别""判断属于哪一类" | 分类/聚类 | K-Means / 逻辑回归 / 决策树 |
| "相关""影响""什么因素决定" | 相关性/因果 | 回归 / 相关系数 / ANOVA |
| "随时间变化""传播""演化" | 动态/微分方程 | SIR模型 / 微分方程 / 元胞自动机 |

## 二、评价类 —— 3 种方法怎么选

```
有没有大量客观数据？
├── 没有（只有主观判断）→ AHP（层次分析法）
│   例：评委打分、专家意见、方案偏好比较
│
├── 有（数据充分）→ 先用熵权法算权重，再用 TOPSIS 排序
│   例：几十个城市的统计指标数据
│
└── 既有主观又有客观 → AHP定权重 + 数据打分
    例：权重来自专家意见，分值来自实际数据
```

## 三、预测类 —— 数据量决定方法

| 数据情况 | 推荐模型 | 原因 |
|---------|---------|------|
| 只有少量数据（<10个点） | 灰色预测 GM(1,1) | 小样本也能用 |
| 有明显趋势和季节性 | ARIMA / 指数平滑 | 经典时间序列 |
| 多变量影响预测目标 | 多元线性回归 | 可解释性强 |
| 数据量大、关系复杂 | 神经网络 / 随机森林 | 拟合能力强 |
| 知道物理/数学关系 | 曲线拟合 + 微分方程 | 机理明确 |

## 四、优化类 —— 约束条件决定方法

```
目标函数和约束都是线性的？
├── 是 → 线性规划（linprog）
│
└── 有整数变量？
    ├── 是 → 整数规划 / 0-1规划
    └── 否（非线性）→
        ├── 能找到解析解 → 非线性规划（fmincon）
        └── 找不到 → 遗传算法 / 模拟退火
```

## 五、比赛中最常用的 5 个模型套路

### 套路1：评价排序（最常用）
```python

# 或者用新工具库
from 竞赛工具库 import normalize

# Step 3: TOPSIS排序
scores = topsis(data, w)
```

### 套路2：数据预测
```python
from 必备模型代码库 import gm11
# 灰色预测（小样本）/ 回归（大样本）
fit, pred = gm11(data, n_pred=5)
```

```python
from scipy.optimize import linprog
# 线性规划求解

```

## 六、论文中必须包含的分析（每篇必写）

| 分析内容 | 对应代码 | 为什么重要 |
|---------|---------|-----------|
| 描述性统计 | data.describe() | 让评委了解数据 |
| 可视化 | matplotlib 图表 | 比文字直观100倍 |
| 误差分析 | regression_metrics() | 证明模型可靠 |
| 灵敏度分析 | sensitivity_analysis() | 验证模型稳定 |
| 模型对比 | compare_models() | 说明选这个模型的原因 |

## 七、时间紧张时别碰的模型

- ❌ 深度学习（CNN/RNN/LSTM）—— 数据量、算力、时间都不够
- ❌ 贝叶斯网络 —— 建模复杂，评委不一定能快速看懂
- ❌ 支持向量机 —— 可解释性差，论文不好写
- ❌ 主成分分析（PCA）—— 降维后维度含义不明确，评委追问不好解释

✅ **优先选择评委一眼能看懂的模型**：线性回归、AHP、TOPSIS、K-Means、线性规划
