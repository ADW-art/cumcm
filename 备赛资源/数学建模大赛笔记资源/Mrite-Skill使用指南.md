## 五、使用流程

### 5.1 赛前准备（比赛前）

确认环境都就位：
```powershell
xelatex --version      # LaTeX正常
python --version       # Python正常
claude --version       # Claude Code正常
```

### 5.2 拿到赛题后（比赛开始）

**步骤 1：放入赛题**

把赛题文件（PDF 或 DOCX 格式）放入 `题目/` 文件夹：
```
E:\code\数学建模大赛\Mrite\题目\A题.pdf
```

**步骤 2：放入附件数据**

把赛题附带的 Excel/CSV/DOCX 数据文件放入 `数据/` 文件夹：
```
E:\code\数学建模大赛\Mrite\数据\附件1.xlsx
E:\code\数学建模大赛\Mrite\数据\附件2.csv
```

**步骤 3：启动 Claude Code**

```powershell
cd E:\code\数学建模大赛\Mrite
claude
```

Claude Code 启动后会读取当前目录下的 `CLAUDE.md` 文件，自动加载 Mrite 技能配置。

**步骤 4：下达指令**

在 Claude Code 的命令行界面中输入：

```
开始求解
```

### 5.3 执行流程（全程自动，无需干预）

输入"开始求解"后，AI 会按以下顺序自动执行：

**阶段 ① —— 读取输入**
- 读取 `题目/` 下的赛题文件（用 PDF 解析器或 DOCX 解析器）
- 读取 `数据/` 下的所有数据文件
- 全量读取数据，不只前几行
- 检查数据完整性：列数是否一致、数据类型是否统一、有无合并单元格等
- 打印数据总览

**阶段 ② —— 生成求解计划**
- 写入 `求解/求解计划.md`
- 包含：题目总体方向、各题求解思路、各题输出标准、操作步骤、文件清单、异常预案

**阶段 ③ —— 逐题求解**
- 每个问题一个 Python 文件，放在 `求解/问题序号/`
- 先计算所有结果，然后画图（每题至少 3-5 张图）
- 自动做灵敏度分析
- 保存结果到 `求解/问题序号/结果/`

**阶段 ④ —— 写论文**
- 按 LaTeX 模板格式逐章撰写
- 每章一个 `.tex` 文件
- 公式、图表、引用自动生成

**阶段 ⑤ —— 编译排版**
- 自动运行 `xelatex 论文.tex` 编译
- 如果编译有错误，自动修复后重新编译
- 优化排版（调整表格宽度、图表位置等）
- 最终输出 `论文/论文.pdf`

**阶段 ⑥ —— 排版优化（自动）**
- 检查 PDF 排版是否溢出
- 自动修复宽度问题
- 直到日志干净、排版正常

### 5.4 流程结束后

最终产物：

| 文件 | 说明 |
|------|------|
| `论文/论文.pdf` | ✅ **最终论文！可以直接提交** |
| `求解/求解计划.md` | 求解思路文档 |
| `求解/问题一/代码.py` | 各问题的求解代码 |
| `求解/问题一/图片/*.png` | 生成的图表 |

## 七、注意事项

### 7.1 论文格式

Mrite 生成的论文遵循国赛标准格式：
- 摘要不超过 900 字 / 1 页
- 正文不使用列表符号（一律自然段落）
- 所有图表必须有标题且在正文中被引用
- 表格不换行，使用 longtable 自动分页

### 7.2 AI 工具使用声明

根据《全国大学生数学建模竞赛人工智能工具使用规定（2025年试行）》：

> 使用 AI 工具的参赛作品必须在支撑材料中包含 AI 工具使用详情说明
> （PDF 格式，以"AI 工具使用详情"为文件名）

内容包括：
- 所用 AI 工具名称和版本
- 具体使用目的和环节
- 关键交互记录（重要提示词与回复）
- 采纳和人工修改情况

模板：
```
[1] Claude, Claude Code, Anthropic, 2026-08-10
```

### 7.3 人工修改的必要性

Mrite 生成的是初稿，提交前建议人工检查以下内容：
- ✅ 摘要是否准确反映了论文内容
- ✅ 公式是否正确（特别是变量符号是否一致）
- ✅ 图表标题是否与内容对应
- ✅ 参考文献格式是否规范
- ✅ 有无错别字和病句

### 7.4 需要补充的字体

如果编译报字体错误，可以修改 `论文/format.cls`，把思源宋体改为系统自带的字体：
```
Windows 系统已有中文字体：微软雅黑、宋体、SimSun 等
```

## 九、遇到问题

| 问题 | 解决办法 |
|------|---------|
| xelatex 编译报错"字体找不到" | 修改 format.cls 把思源宋体改为系统字体 |
| Python 报错 ModuleNotFoundError | 运行 pip install chardet openpyxl xlrd |
| Claude Code 无法连接 | 检查网络/代理设置（DevSidecar 可能拦截） |
| 生成的 PDF 内容超出页面 | AI 会自动修复，如未修复可提示"优化排版" |
| 需要手动调整某章节 | 直接修改 论文/*.tex 后重新 xelatex 编译 |
